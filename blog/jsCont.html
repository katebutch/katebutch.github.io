More JavaScript

Blocked

This week was by far the most challenging sprint. I found JS fundamentals very challenging last week and didn't enter sprint 4 with a lot of confidence. I struggled most of all in katas.

Most of my problem solving in this was heavily reliant on googling problems, breaking what I was trying to do into chunks, defining each thing and really trying to get a grip on exactly what I was trying to do. Once I had that I could check if I had all my ducks, then put them in a row. I found it a lot easier to figure out where I went wrong when I had some level of confidence that I knew all the elements I needed. I could follow that with diagnosing my syntax issues. Primarily this came from researching particular elements, but I found a lot of valuable help in the discord from other people going through the same issues. Error messages were also helpful most of the time when they were specific, but I had to have my code fundamentally in-check to receive helpful error messages.

How did I feel? Pretty low and defeated for most of the sprint. When I did have breakthroughs I was still second-guessing my abilities and did not feel confident to explain the solution I had come up with.

I learned a few things this week. One was straight away in katas - when something is defined you don't have to declare or do anything to the object (like, function.. 'let') to change something about it. You can just say, hey thing, you're this now. I did begin to form an understanding of how to separate out your code and then condense it into an ES6 style of code. For some reason ES6 has seemed much more comfortable to me - maybe because I still feel intimidated by large chunks of code.

'elegantly solved a problem'

I was more relaxed with JS-Gradebook. The first problem I encountered was to think about how to access an array inside of an array and push information into it. I recalled the 'push' method, googled it to make sure my instinct was right and to double check the syntax of how to use a method in code. I broke it down into logical steps using pseudocode and attempted different ways of writing out the code. My first few attempts failed, so I consulted Joseph's video and found that I had used wrong brackets (square for students and score). In this I learned that we use [] for functions that we have declared prior but within an object - so it grabs information from inside an object and () for posing an argument to functions (or in this case, a method) that already exist in the array.
